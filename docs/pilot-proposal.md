# EduCommand 60-Day Pilot Proposal

## 1. What EduCommand Is

EduCommand is a simple command dashboard for school principals, coordinators, and teachers.

It helps school leadership track syllabus coverage, teacher updates, delegated tasks, school events, delays, and institutional readiness from one place.

EduCommand is not a full school ERP. It is focused on daily academic and operational visibility for the principal.

## 2. What Problem It Solves

In many schools, important updates are scattered across WhatsApp messages, verbal follow-ups, notebooks, and last-minute reports.

This creates common challenges:

- Syllabus progress is difficult to track class-wise and subject-wise.
- Tasks are assigned verbally and follow-up becomes inconsistent.
- Events create last-minute pressure because readiness is not visible early.
- Teachers submit updates in different formats or only when reminded.
- Reports are prepared only when management or inspection teams ask for them.

EduCommand gives the principal a clear weekly view of what is on track, what is delayed, and what needs attention.

## 3. Pilot Scope

The 60-day pilot will be implemented for one school campus.

The pilot will cover selected classes, staff members, subjects, school events, and internal tasks. The goal is to test whether EduCommand improves visibility, follow-up discipline, and readiness during a normal school working cycle.

## 4. Modules Included

The pilot includes the following EduCommand modules:

- Syllabus coverage tracker
- Task delegation tracker
- Calendar and event readiness
- Weekly teacher updates
- Delay alerts
- Institution brief generator

## 5. What the School Needs to Provide

To set up the pilot, the school will need to provide:

- Staff list
- Class and section list
- Subject list
- Teacher assignments
- Upcoming events

The school may provide this information in Excel, Google Sheets, or any simple document format.

## 6. Pilot Duration

The pilot duration will be 60 days.

During this period, the school can use EduCommand for weekly academic tracking, task follow-up, event readiness, and principal review.

## 7. Success Metrics

The pilot will be evaluated using the following success metrics:

- Weekly update compliance: percentage of teachers submitting weekly syllabus updates on time.
- Overdue task reduction: reduction in pending and overdue delegated tasks.
- Syllabus visibility: principal can clearly see class-wise and subject-wise coverage.
- Event readiness improvement: major events have owners, milestones, and readiness status before the event date.
- Principal satisfaction: principal feedback on usefulness, simplicity, and daily relevance.

## 9. Implementation Timeline

Suggested implementation plan:

| Week | Activity |
| --- | --- |
| Week 1 | Collect school data, create school profile, create user accounts, configure classes, subjects, and assignments |
| Week 2 | Train principal, coordinators, and selected teachers; start weekly syllabus updates and task tracking |
| Weeks 3-4 | Monitor usage, review weekly update compliance, track tasks and events |
| Weeks 5-7 | Improve follow-up rhythm, use institution brief and reports for review meetings |
| Week 8 | Review pilot results, collect feedback, and decide next steps |

## 10. Training Plan

Training will be simple and role-based:

- Principal training: dashboard review, delay alerts, reports, institution brief, and weekly workflow.
- Coordinator training: syllabus review, task creation, event readiness, and milestone follow-up.
- Teacher training: submitting weekly syllabus updates, updating assigned tasks, and updating event milestones.

Training can be completed through one short orientation session and one follow-up support session.

## 11. Limitations

EduCommand is focused on school command visibility and institutional readiness.

For this pilot, EduCommand is:

- Not a fee ERP
- Not a parent app
- Not a student attendance system

These areas can remain in the school’s existing systems.

## 12. Closing Note

EduCommand is designed to help principals run the institution with better weekly visibility and less manual chasing.

The 60-day pilot gives the school a low-risk way to test whether a simple command dashboard can improve syllabus tracking, staff accountability, event readiness, and institutional reporting.
